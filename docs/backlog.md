# Backlog

## Purpose
This document stores ideas, possibilities, and uncommitted design directions.

Use this file for:
- brainstorming results
- interesting ideas not yet accepted into core design
- risky or uncertain possibilities
- things worth remembering but not yet committing to

Items here are not design truth unless promoted into the proper design documents.

---

## High-Priority Ideas
These are ideas that seem promising and may deserve testing or future commitment.

- gate milestone reward thresholds
- gate exploration points of interest outside the safe objective zone
- multiple reward categories within gates
- unique materials for weapon and structure variety
- components used for turret and trap customization
- extraction pressure spike as a defining gate mechanic

---

## Needs Discussion
These are important questions that may significantly influence design.

- should gate worlds have persistent progression between visits
- should players be able to start at a previously reached gate threshold
- should gate progression feel more like survival, exploration, or a hybrid
- how much building should be allowed inside gates
- should resources be fully shared in co-op by default
- what is the best failure penalty for gate runs

---

## Could Be Interesting Later
These are lower-priority ideas that may be good once the core loop works.

- gate biome mutators
- weather or environmental hazards
- temporary gate deployables
- gate-specific bosses or event triggers
- local persistent gate-region state
- support-oriented player tools
- structure component sockets or modifications

---

## Cool but Dangerous
These ideas may be exciting but risky for scope or clarity.

- highly persistent procedural gate worlds
- deep gate-world progression layers
- full open-world exploration inside gates
- large crafting trees for materials
- very large number of unique resource categories
- complicated trap systems too early

## Rules
Add ideas here when:
- they are interesting but not yet confirmed

Do not use this file for:
- implementation progress
- polishing notes
- confirmed project truth that belongs in specs/GDD
