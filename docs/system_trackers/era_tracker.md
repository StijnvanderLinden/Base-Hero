# Era System Tracker

## Current Status
Stone Age Only For MVP

## Current Design Summary
The era system should stay lightweight during the vertical slice.

Stone Age is a theme and content wrapper for the first arena. Multiple eras and deeper era progression are backlog.

## Implemented
- EraData resource class exists
- EraManager exists
- Stone Age data hooks exist

## In Progress
- Keeping era data useful without expanding scope

## Blockers / Problems
- Existing era data may still reference materials, research, or broader content that is out of MVP scope

## Must Have
- One Stone Age-flavored arena slice
- Data hooks only where they directly support the MVP

## Should Have
- Clear enough naming/presentation that the first slice has identity

## Could Have
- More era presentation later

## Won't Have (for now)
- Multiple playable eras
- Era unlock chains
- Era-specific material trees
- Complex era-selection UI

## Open Questions
- What is the minimum Stone Age theme needed for readability?

## Recent Decisions
- Multiple eras are deferred until the first arena loop is fun

## Next Recommended Task
Keep Stone Age focused on the arena loop: enemies, turrets, scrap, essence, and repeatability.
